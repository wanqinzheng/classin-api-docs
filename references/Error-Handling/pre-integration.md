# 对接之前常见问题



##  开通API对接是否需要收费？
这个跟机构开通的时候购买的服务版本有关，具体请参考 https://www.eeo.cn/cn/pricing 


##  对接API有什么难度吗？ 对接API需要多久？
对接 ClassIn API 的前提是合作机构具备相应的开发能力，通过调用 API 可实现机构自有平台与 ClassIn 系统的数据同步，具体对接方案如下：      
- 若机构现有系统未配备排课功能，需先行开发排课模块，再基于该模块完成与 ClassIn API 的对接。
- 若机构已具备成熟的排课功能，仅需将自有平台的排课逻辑与 ClassIn 的班级、课节逻辑进行匹配，调用对应接口即可完成数据同步。

ClassIn API 提供了丰富且轻量化的接口能力，最快对接周期在10-15个工作日。针对机构的个性化需求，我们也支持定制化的全功能对接方案。   


##  一个ClassIn账号可以对接多个平台吗
从技术实现层面而言，单个 ClassIn 账号支持与多个外部平台进行对接，支持多平台同时通过 API 发起排课请求，或同步使用 API 消息订阅功能完成消课操作。   

但从业务管理视角出发，不建议采用此种对接模式。    
因为多平台并行调用会导致数据链路复杂度大幅提升，不仅难以精准追溯数据来源，还会对后续的运营管理与问题排查造成阻碍。

## ClassIn可以和 XX系统对接吗？需要什么条件
可以的。ClassIn提供了标准的API接口，任何平台都可以调用接口，开发相应的排课功能，将平台数据包括老师学生信息和排课数据同步到ClassIn。从而实现两个系统对接打通。

需要平台技术团队使用ClassIn接口进行开发，实现自动同步。


## 可以对接小程序约课平台吗？需要小程序供应商做什么？
可以的。学生通过小程序约课，然后小程序调用API把约课数据同步到ClassIn。学生就能实时在ClassIn看到自己约的课了。      
需要小程序开发方调用ClassIn接口进行开发，实现自动同步。

##  API对接后，可以实现从小程序跳转到ClassIn吗
小程序里只能通过让用户复制链接到浏览器打开实现跳转。这是微信的限制，小程序不支持跳到任何APP的。

##  怎么获取SID和秘钥？
如果已经开通API  请登录机构后台，在我的机构--API对接处查看 

## 开通了API，可以不用下载ClassIn App吗？或者可以定制App logo、风格吗？
- API仅有数据同步功能，开通了API，仍然需要使用ClassIn App，也无法对App进行定制。
- 如果使用网页大直播方案，可以实现学生用网页端观看，或者将网页内嵌到客户app里的效果，但是老师端仍需要用ClassIn。 
具体参考 https://docs.eeo.cn/api/zh-hans/Solutions/Live&PlaybackScenes.html 

##  ClassIn已经对接的系统包括哪些？
ClassIn目前已经跟多个成熟的平台进行了对接，使用已对接平台的客户，可以自动与ClassIn实现排课和消课的打通。 
具体功能包括：
1.  老师和学生的同步（手机号）
1.  排课信息的同步（包括每一堂课上课的时间、老师和学生），
    - 课堂支持设置是否录课、直播回放
    - 课堂坐席1vN这个设置，每个平台做法不一，可能固定1v6或者提供选项。请参考平台手册。     
    - 学生对接为班级学生或者插班生。具体参考平台手册，比如校宝对接为插班生。
    - 更多设置需要到ClassIn后台配置。比如联席老师、课节资源等。
1.  出勤信息同步：部分平台对接了出勤信息，可以实现自动消课，包括好约课、学邦、校管家和校宝等

##  想实现机构的校管家/学邦等平台跟ClassIn打通应该怎么做？
ClassIn已于多个教务平台实现打通，每个平台打通的功能在细节上略有差异，建议先阅读平台的功能说明后，再进行打通。打通一般步骤如下：   
1. 跟教务平台商务沟通对接事宜，确定平台支持打通。  
1. 在ClassIn购买相应的服务版本并开通API。
1. 开通API 后，请登录机构后台查看SID和秘钥并配置到教务平台。
1. 部分平台还需要提供一个消息订阅地址给ClassIn，配置消息推送。在ClassIn支持群，提供平台地址，配置消息类型（这两个由平台提供），SID和一个邮箱用于接收报错信息
1. 创建测试课程和学生，验证对接效果是否满足需求。  
1. 正式使用。
